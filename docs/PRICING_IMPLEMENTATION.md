# Pricing & Credit System Implementation

## Overview

This document describes the implementation of the credit-based pricing system with three tiers: Free, Starter, and Pro.

## Pricing Model

### Plans

| Plan    | Price | Credits/Month | Cost per Credit | Features                    |
|---------|-------|---------------|-----------------|----------------------------|
| Free    | $0    | 100           | Free            | Watermarked PDFs           |
| Starter | $9    | 1,000         | $0.009          | No watermark, overage option |
| Pro     | $25   | 5,000         | $0.005          | Priority queue, overage option |

### Overage Pricing
- Starter: $0.009 per credit
- Pro: $0.005 per credit
- Must be manually enabled in Settings
- Not available for Free tier

## Database Schema

### Updated `user_profiles` Table

```sql
CREATE TABLE public.user_profiles (
    id UUID PRIMARY KEY,
    tier TEXT NOT NULL DEFAULT 'free' CHECK (tier IN ('free', 'starter', 'pro')),
    monthly_credits INTEGER NOT NULL DEFAULT 100,
    credits_used INTEGER NOT NULL DEFAULT 0,
    overage_enabled BOOLEAN NOT NULL DEFAULT false,
    subscription_period_start TIMESTAMPTZ DEFAULT NOW(),
    stripe_customer_id TEXT,
    subscription_status TEXT,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

### Security

Row Level Security (RLS) policies ensure that:
- Users can only read their own profile data
- Users can ONLY update `overage_enabled` field
- Credits and tier can ONLY be modified by service role (internal endpoints)

## API Endpoints

### User-Facing Endpoints

#### GET /api/v1/user/profile
Returns complete user profile including credits, usage, and settings.

**Response:**
```json
{
  "profile": {
    "email": "user@example.com",
    "tier": "starter",
    "monthly_credits": 1000,
    "credits_used": 150,
    "credits_remaining": 850,
    "overage_enabled": false,
    "subscription_period_start": "2024-01-01T00:00:00Z"
  }
}
```

#### PATCH /api/v1/user/settings
Update user-controllable settings (overage toggle only).

**Request:**
```json
{
  "overage_enabled": true
}
```

#### POST /api/v1/user/checkout
Initiate plan upgrade (placeholder for Stripe integration).

#### DELETE /api/v1/user/account
Permanently delete user account and all associated data.

### Internal/Admin Endpoints

Require `x-service-key` header with `SUPABASE_SERVICE_ROLE_KEY`.

#### POST /internal/billing/update-plan
Update user's plan and credits.

**Request:**
```json
{
  "user_id": "uuid",
  "tier": "pro",
  "monthly_credits": 5000
}
```

#### POST /internal/billing/increment-usage
Increment user's credit usage (called internally by job creation).

**Request:**
```json
{
  "user_id": "uuid",
  "credits": 1
}
```

#### POST /internal/billing/reset-credits
Reset monthly credits for all users (for cron jobs).

## Job Creation with Credit Checking

When creating a job via `POST /api/v1/jobs`:

1. **Check credits**: Verifies user has remaining credits or overage enabled
2. **Create job**: If credits available, job is created
3. **Increment usage**: Credits_used is incremented by 1
4. **Return info**: Response includes credit usage information

**Response with credits:**
```json
{
  "jobId": "uuid",
  "status": "pending",
  "outputType": "pdf",
  "message": "Job created successfully",
  "credits": {
    "used": 151,
    "remaining": 849,
    "monthly_limit": 1000
  }
}
```

**Error when limit reached:**
```json
{
  "error": "Credit limit reached",
  "details": {
    "monthly_credits": 100,
    "credits_used": 100,
    "credits_remaining": 0,
    "overage_enabled": false,
    "message": "You have used all your monthly credits..."
  }
}
```

## Watermarking

PDFs generated by Free tier users automatically include a watermark.

**Implementation:**
- Checked in `services/pdfService.js`
- Watermark added only if `account.tier === 'free'`
- Starter and Pro tiers: no watermark
